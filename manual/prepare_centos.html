Prepare CentOS 7 for SMS Server Installation
1. Minimum Resources Required
- Intel i5 2.5Ghz CPU
- 2GB RAM
- 100GB free space (for 50 active users)
- 1Mb/s internet connection (fixed or dynamic IP address)
 
2. Preparation Steps
Select 'Install CentOS 7' option and press enter

Figure 1
 
After select your desired language, then setup page 'INSTALLATION SUMMARY' will be displayed. You should cast your server here by selecting system time zone, input network connection parameters, configure hard disk partitions, etc.

Figure 2
 
Configure network settings, select 'NETWORK & HOST NAME'. Then click 'Configure...' button, and then select 'IPv4 Settings' and click 'Add' button. Input network settings of your server, and turn on the network connection afterward.

Figure 3
 
Configure your server's time zone, and turn on network time auto-adjustment. Note: SMS is a single time zone application, and time zone is very important for it's operation. For example, if you want to provide SMS service for users in Hong Kong, you must set your server to Hong Kong time zone, even it is not located in Hong Kong physically.

Figure 4
 
Select the installation destination and configure your partitions manually like Figure 6 shown. Although you may let CentOS configure disk partitions for you automatically, CentOS will assign half disk space to 'home' partition, which SMS will not use it. i.e. About half of disk space will be wasted.

Figure 6
 
Security policy is varied between different CentOS 7 builds, and you should select standard or default policy for SMS server installation. Now, button 'Begin Installation' on figure 2 should be activated and you could click on it to install CentOS 7. During CentOS 7 installation, two more tasks you should do: (1) Assign root password (blank password for root is totally unacceptable), and (2) Create another administrative user. Just click on 'ROOT PASSWORD' and 'USER CREATION', new screen will be displayed and follow the instructions to create root password and a new administrative user. (Figure 8)

Figure 7
 

Figure 8
 
After CentOS 7 installation is completed, reboot the system. Then two things must be done immediately to harden the system security: (1) disable root login via ssh, (2) patch the system to the most updated level.

For (1), modify the ssh configuration file "/etc/ssh/sshd_config", and uncomment the option "PermitRootLogin yes" and change it to "PermitRootLogin no". Then, save the change and restart the sshd service:

systemctl restart sshd.service

For (2), issue the below command on a terminal:

yum update

Now, the system is well prepared for SMS installation. Reboot the system to ensure all most updated patches loaded.
