SMS Server Installation Guide
1. Prepare SMS server installation package
- Download SMS server installation package from GitHub either:
(1) Use Git to clone the project on your computer by issue command on a terminal: git clone https://github.com/dw999/sms.git
 
(2) Download a zipped archive named sms_master.zip from https://github.com/dw999/sms, and then decompress it with command: unzip sms_master.zip
 
- Make sure all installation shell scripts are executable: chmod +x *.sh and chmod +x ffmpeg/*.sh
 
2. Prerequisites of SMS server installation
- You are root or root equivalent user, i.e. You must be super user.
- Network connection of your CentOS or Ubuntu Linux server is well configured and fast enough (> 1Mb/s).
- You have registered two domain names for the decoy site and messaging site, and they have already been pointed to your server's public IP address.
- You have an email address for the default SMS administrator. (Note: It should not link to your true identity)
- You have registered at least one Gmail account for SMS operations. (Note: It MUST be Gmail account) However, if Gmail is blocked by your local government, see Figure 11 for work-around.

Note:
This guide is based on CentOS 7 environment, but it could also be applied to other supported platforms.
 
3. Install SMS server
If all prerequisites of SMS server installation are fulfilled, you may now install SMS server.

Login Linux server as super user, open a terminal window, and go to the directory where decompressed SMS server installation packages located, then execute installation script:

# ./install_sms.sh

Screen like Figure 1 will be displayed. You may now press enter to start SMS server installion.

Figure 1
 
Installation process is divided into 10 steps, they are:
Step 1: Install required applications
Step 2: Install required Perl libraries
Step 3: Get required Javascript libraries
Step 4: Copy program files and Javascript libraries to pre-defined directories
Step 5: Prepare database server and create databases
Step 6: Input essential data to SMS
Step 7: Install SSL certificates to the sites
Step 8: Configure the Linux system settings
Step 9: Build audio file converter FFmpeg (optional)
Step 10: Install system defender (optional)

After you press the enter, step 1 will proceed, and screen like Figure 2 will be seem.

Step 6 is very important, you need to give the system seven essential data for your SMS server. They are SMS connection mode, DNS name of the decoy site, DNS name of the messaging site, email address of the SMS server administrator, Gmail address for SMS operations, the Gmail account login username and it's password. Any incorrect input data will let SMS server malfunction.

Please also note that you MUST put 'https://' in front of your DNS names of the decoy site and messaging site, just as Figure 11 shown.

After you input all these data, you require to verify and confirm them. If all input data are correct, press 'Y' and then enter to save them. Otherwise, press 'N' to input them again.

Figure 11
 
Step 7 will generate SSL certificates in accordance with DNS names of your decoy site and messaging site you input in step 6. In this step, you will be asked to input your email address (just use the same email for the default SMS administrator), and three questions. Figure 12 provides you an example.

Note:
This step may sometimes fail (Figure 13), but don't worry, you can repeat this process later after installation by running the below command to subscribe SSL certificates again:

# certbot --apache

SSL certificates are used to provide the secrecy of your online communication, which is the most important part of the SMS server. Therefore, you must ensure your SSL certificates installation successful before using SMS server. You could check how secure of your web sites by using testing site https://www.ssllabs.com:

The general format is https://www.ssllabs.com/ssltest/analyze.html?d={your_web_site_dns_name}

e.g.) https://www.ssllabs.com/ssltest/analyze.html?d=argogo.mywire.org

It will give security ratings for your site (similar to Figure 14). If grade of overall rating is below A, that means something is wrong of your sites, please fix all the issues before using SMS server for your messaging operation.

After running step 8 to configure some Linux system settings, you will process to optional step 9. You will be asked to build multimedia file converter FFmpeg, it is used to convert some common audio input file formats used by Android to OGG audio file format, which is widely supported by HTML5 multimedia objects in many platforms (except iOS). If you don't have it, your audio input files will be treated by SMS server as file attachment. You must download them to your phone or computer to play their content, but can't play on your web browser directly. Therefore, for a better user experience, it is worth to spend some time to build FFmpeg.

Figure 15
 
The next optional installation item is system defender. It is not the necessary component to drive SMS server, but it provides minimum protection to your SMS server from cyber criminals. So, it is better to have it. The installation program needs to create database for system defender, you will be asked to input the database server administrator password here.

Figure 16
